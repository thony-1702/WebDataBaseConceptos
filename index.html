Bases de Datos

Conceptos fundamentales visualizados

1. Estructura Básica

Tabla

Contenedor principal de datos sobre un tema (ej: Clientes). Es la estructura donde se organiza la información.

Columna (Campo)

Es la unidad vertical. Define el tipo de dato (texto, número, fecha) que se almacenará. Representa los atributos del objeto.

Fila (Registro)

Es la unidad horizontal. Representa una instancia única o un ítem individual dentro de la tabla (ej: Un cliente específico).

2. Llaves (Keys)

Primary Key (PK)

Es el identificador único de cada fila. Garantiza que no haya registros duplicados. No puede ser nulo (vacío).

Foreign Key (FK)

Es un campo que apunta a la Primary Key de otra tabla. Es el "puente" que permite conectar dos tablas diferentes.

3. Tipos de Relaciones

1 : 1

Uno a Uno

Concepto: Un registro en la Tabla A se relaciona con un solo registro en la Tabla B, y viceversa.

En la BD: Normalmente comparten la misma Llave Primaria o se coloca la Llave Foránea en cualquiera de las dos tablas con restricción "Unique".

Ejemplo: Un empleado tiene un único archivo de antecedentes penales.

1 : N

Uno a Muchos

Concepto: Un registro en la Tabla A (Padre) se relaciona con múltiples registros en la Tabla B (Hijo).

En la BD: Es la relación más común. La Llave Foránea SIEMPRE se coloca en la tabla del lado "Muchos" (Tabla B).

Ejemplo: Un cliente realiza muchos pedidos. (El ID del cliente se guarda en cada pedido).

N : M

Muchos a Muchos

Concepto: Varios registros de A se relacionan con varios de B.

En la BD: ¡No se puede hacer directamente! Se requiere crear una tercera tabla llamada Tabla Intermedia (o Pivote) que contenga las Llaves Foráneas de ambas tablas originales.

Ejemplo: Estudiantes y Cursos. Se crea la tabla "Inscripciones" para unirlos.
